3) Permissions before and after modification
Command:
ls -ld secure_workspace secure_workspace/*
Output:
drwxrwxrwx+ 5 codespace codespace 4096 Jul 3 17:19 secure_workspace
drwxrwxrwx+ 2 codespace codespace 4096 Jul 3 17:19 secure_workspace/docs
drwxrwxrwx+ 2 codespace codespace 4096 Jul 3 17:19 secure_workspace/logs
drwxrwxrwx+ 2 codespace codespace 4096 Jul 3 17:19 secure_workspace/src

Command:
chmod 750 secure_workspace
chmod 750 secure_workspace/docs secure_workspace/src secure_workspace/logs
chmod 640 secure_workspace/docs/plan.txt secure_workspace/src/app.txt secure_workspace/logs/activity.log

Command:
ls -ld secure_workspace secure_workspace/*
Output:
drwxr-x---+ 5 codespace codespace 4096 Jul 3 17:19 secure_workspace
drwxr-x---+ 2 codespace codespace 4096 Jul 3 17:19 secure_workspace/docs
drwxr-x---+ 2 codespace codespace 4096 Jul 3 17:19 secure_workspace/logs
drwxr-x---+ 2 codespace codespace 4096 Jul 3 17:19 secure_workspace/src

Command:
ls -l secure_workspace/*
Output:
secure_workspace/docs:
-rw-r----- 1 codespace codespace 0 Jul 3 17:19 plan.txt

secure_workspace/logs:
-rw-r----- 1 codespace codespace 0 Jul 3 17:19 activity.log

secure_workspace/src:
-rw-r----- 1 codespace codespace 0 Jul 3 17:19 app.txt

Observation:
The directory permissions were tightened from full access to owner-only access and the files were restricted to read-only access for group and others.

4) Ownership and umask
Command:
stat -c '%n %U %G %a' secure_workspace secure_workspace/docs secure_workspace/src secure_workspace/logs secure_workspace/docs/plan.txt secure_workspace/src/app.txt secure_workspace/logs/activity.log
Output:
secure_workspace codespace codespace 750
secure_workspace/docs codespace codespace 750
secure_workspace/src codespace codespace 750
secure_workspace/logs codespace codespace 750
secure_workspace/docs/plan.txt codespace codespace 640
secure_workspace/src/app.txt codespace codespace 640
secure_workspace/logs/activity.log codespace codespace 640

Command:
umask
Output:
0022

Observation:
Ownership remained with the current user and the default umask value was 0022, which is commonly used for a secure and standard Linux setup.

5) Why these permissions matter
The 750 permissions on the directories allow only the owner to enter and manage the workspace, while group members and others are denied access. The 640 permissions on the files protect the content from general read access and allow only the owner to modify them. This setup helps reduce accidental exposure of project data.
